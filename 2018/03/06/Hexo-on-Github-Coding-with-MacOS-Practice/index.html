Hexo on Github+Coding with MacOS Practice

发表于 2018-03-06 | 分类于 Hexo |

使用Hexo静态博客部署在github及同类网站上的好处：

  • 自由，没有广告
  • 简单，全静态页面，部署后常用操作不超过一只手
  • 快速，据说速度是jekyll的N倍
  • 稳定，只要Github存在，你的博客就没事
  • 免费，不需要云服务器的费用
  • 强大，支持一次编辑，多处发布
  • 标准，支持Markdown语言的解析
  • 扩展，支持很多扩展包、主题包，想换就换

¶遇坑总结

  • 空格

    不管是_config.yml配置还是“title, date, tags”这几个文章标题项，冒号后面需要空格，否则会报错。

    另外其实Markdown文本里，空格也非常重要，比如###作为三级标题，后面也需要有个空格，否则不能正常展示。

  • SSH Key

    这个Key的生成和配置是和邮箱绑定的，另外在配置链接时，有https和ssh两种方式，两种方式的配置是不一样的，如下，如果搞不清楚，就会发生『我的SSH KEY设置了，怎么还不生效』这样的疑问：

    除了配置多个SSH Key，其实还有一个简单的解决方案，就是通过github邀请加入项目，这样，一个Key可以同步多个不同的github账号及其项目，邀请的关键在于『通过邮箱确认』，详见这篇博文：http://blog.csdn.net/tczzyzymj/article/details/53871763

    重要部分节选如下：

  • 邀请小伙伴加入：

1、双方都有GITHUB帐号（废话……）

2、在你的项目中找到Settings，在Settings下面找到Collaborator，点击Collaborator

3、输入你的小伙伴的用户名，上面会说，输入用户名、完整名字或者email地址，但是我只有输入了我小伙伴的用户名才找到！

4、当Add Collaborator可以点击的时候，点击就可以邀请小伙伴了

5、告诉你的小伙伴，让他去他的注册邮箱里面找一封确认信！！！可能被标记为垃圾邮件了！一定要确认，否则无法完成！

6、完成，大家可以愉快的提交代码了！

  • 语言的配置有两处，博客目录的_config.xml，和主题目录的同名文件，里面都有**language:**属性，需要同时设置才能生效。
